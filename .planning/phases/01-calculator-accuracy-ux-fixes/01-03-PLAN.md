<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Binance API → Browser | Price data arrives over HTTPS from fapi.binance.com; no user authentication required |
| API price → Input fields | Price value from API is rendered directly into input fields when auto mode is ON |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | Binance Futures price feed (fapi.binance.com) | accept | App uses HTTPS to fapi.binance.com public endpoint. Price is display-only in a personal tool; no funds at risk from spoofed display value. User retains manual override. |
| T-03-02 | Information Disclosure | currentPrice displayed in input fields | accept | Price data is public market data. No PII or secrets exposed. |
| T-03-03 | Tampering | Auto-price overwrites user input in TP/SL fields | mitigate | Fields only update when toggle is explicitly set to ON. User can set any field to Manual mode to freeze value. Toggle state is not reset by other interactions. |
| T-03-04 | Denial of Service | useEffect dependency array causing infinite re-render | mitigate | Do not call handler functions inside the effect unless price value actually changed (all checks use !== comparison). handleTakeProfitChange is useCallback-wrapped in useCalculator. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new packages. Changes are pure React JSX and TailwindCSS classes. |
</threat_model>

## Critical Anti-Patterns (do NOT repeat these)
- **NON-USECALLBACK HANDLERS IN EFFECT DEPS**: Never include plain handler functions (defined without `useCallback`) in a `useEffect` dependency array. They are recreated every render → new reference → effect fires every render → infinite loop if the effect calls `setState`. Either wrap handlers in `useCallback` with correct deps, or omit them from the dep array (add eslint-disable comment explaining why).
